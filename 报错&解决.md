## 二、下载脚本（直接可用）

请在IPG目录下新建一个文件：

```bash
download_models.py
```

内容如下：

```python
import os
from huggingface_hub import snapshot_download

# ====== 路径配置（强烈建议用数据盘） ======
BASE_DIR = "/root/autodl-tmp"   # ⭐ 改成你的大盘

SD_PATH = os.path.join(BASE_DIR, "stable-diffusion-v1-5")
VAE_PATH = os.path.join(BASE_DIR, "sd-vae-ft-mse")

# HuggingFace repo
SD_REPO = "runwayml/stable-diffusion-v1-5"
VAE_REPO = "stabilityai/sd-vae-ft-mse"


def download_sd():
    print(f"\n📥 Downloading Stable Diffusion → {SD_PATH}")

    snapshot_download(
        repo_id=SD_REPO,
        local_dir=SD_PATH,
        local_dir_use_symlinks=False,
        resume_download=True,

        # ⭐⭐⭐ 只保留 diffusers 必要文件
        allow_patterns=[
            "unet/*",
            "vae/*",
            "text_encoder/*",
            "tokenizer/*",
            "scheduler/*",
            "model_index.json"
        ]
    )

    print("✅ Stable Diffusion 下载完成")


def download_vae():
    print(f"\n📥 Downloading VAE → {VAE_PATH}")

    snapshot_download(
        repo_id=VAE_REPO,
        local_dir=VAE_PATH,
        local_dir_use_symlinks=False,
        resume_download=True
    )

    print("✅ VAE 下载完成")


if __name__ == "__main__":
    os.makedirs(BASE_DIR, exist_ok=True)

    download_sd()
    download_vae()

    print("\n🎉 所有模型已下载完成！")
```

# 日志记录工具mflow

## 逐字段解析

```yaml
logging:
  log_with: "mlflow"          # 使用什么日志工具
  # log_with: null            # 被注释掉的备选项（不使用任何日志工具）
  project_dir: "mlruns"       # 日志存储目录
  run_name: "ipg_train"       # 本次实验运行的名称
```

---

## 1. `log_with: "mlflow"`

### 含义
指定使用 **MLflow** 作为实验追踪工具。

### MLflow 是什么？

MLflow 是一个开源的**机器学习实验管理平台**，主要功能：

```
┌─────────────────────────────────────────┐
│              MLflow 功能                 │
├──────────────┬──────────────────────────┤
│ Tracking     │ 记录参数、指标、模型文件   │
│ Projects     │ 打包可复现的实验代码       │
│ Models       │ 管理和部署模型            │
│ Registry     │ 模型版本管理              │
└──────────────┴──────────────────────────┘
```

### 实际效果

训练过程中会自动记录：

| 记录内容 | 示例 |
|----------|------|
| **参数** | learning_rate=0.008, batch_size=64 |
| **指标** | loss=0.35, mAP=0.89, Rank-1=0.95 |
| **文件** | 模型权重、配置文件、图片 |
| **元信息** | 运行时间、Git版本、环境信息 |

### 代码中的使用方式（推测）

```python
# 通常配合 HuggingFace Accelerate 使用
from accelerate import Accelerator

accelerator = Accelerator(
    log_with="mlflow",           # ← 对应配置中的 log_with
    project_dir="mlruns"         # ← 对应配置中的 project_dir
)

# 初始化追踪
accelerator.init_trackers(
    project_name="ipg_train",    # ← 对应配置中的 run_name
    config=training_config       # 自动记录所有超参数
)

# 训练循环中记录指标
accelerator.log({
    "loss": loss.item(),
    "mAP": map_score,
    "rank1": rank1_acc
}, step=global_step)

# 训练结束
accelerator.end_training()
```

## 2. `# log_with: null`

### 含义
这是一行**被注释掉的备选配置**，表示不使用任何日志工具。

```yaml
# 使用 MLflow（当前生效）
log_with: "mlflow"

# 不使用日志工具（当前被注释）
# log_with: null
```

### 可选的日志工具

| 值 | 工具 | 说明 |
|----|------|------|
| `"mlflow"` | MLflow | 本地/远程实验追踪 |
| `"tensorboard"` | TensorBoard | Google 的可视化工具 |
| `"wandb"` | Weights & Biases | 云端实验追踪 |
| `"comet_ml"` | Comet ML | 云端实验追踪 |
| `null` | 无 | 不记录任何日志 |

### 切换方式

```yaml
# 方式1：使用 MLflow
log_with: "mlflow"

# 方式2：使用 TensorBoard
log_with: "tensorboard"

# 方式3：使用 W&B
log_with: "wandb"

# 方式4：关闭日志（调试时常用）
log_with: null
```

---

## 3. `project_dir: "mlruns"`

### 含义
MLflow 日志文件的**本地存储目录**。

### 目录结构

运行训练后，会自动生成如下目录：

```
mlruns/
├── 0/                              # 默认实验 ID
│   ├── meta.yaml                   # 实验元信息
│   ├── <run_id_1>/                 # 第一次运行
│   │   ├── meta.yaml
│   │   ├── params/                 # 超参数
│   │   │   ├── learning_rate
│   │   │   ├── batch_size
│   │   │   └── ...
│   │   ├── metrics/                # 指标
│   │   │   ├── loss
│   │   │   ├── mAP
│   │   │   └── rank1
│   │   ├── artifacts/              # 产出文件（模型、图片等）
│   │   │   ├── model.pth
│   │   │   └── config.yaml
│   │   └── tags/                   # 标签
│   │       └── mlflow.runName      # → "ipg_train"
│   │
│   └── <run_id_2>/                 # 第二次运行
│       └── ...
│
└── .trash/                         # 已删除的运行
```

### 查看日志

```bash
# 启动 MLflow UI 查看实验结果
cd /your/project/root
mlflow ui --backend-store-uri mlruns

# 浏览器打开
# http://localhost:5000
```

## 4. `run_name: "ipg_train"`

### 含义
本次实验运行的**名称标识**，方便在多次实验中区分。

### 命名建议

```yaml
# 不同实验使用不同名称
run_name: "ipg_train"              # IPG 方法训练
run_name: "ipg_train_lr0.01"       # 调整学习率
run_name: "baseline_no_ipg"        # 不使用 IPG 的基线
run_name: "ipg_train_epoch200"     # 增加训练轮数
```

## 总结

| 配置项 | 值 | 作用 |
|--------|------|------|
| `log_with` | `"mlflow"` | 使用 MLflow 追踪实验 |
| `project_dir` | `"mlruns"` | 日志存储在 `./mlruns/` 目录 |
| `run_name` | `"ipg_train"` | 本次运行命名为 "ipg_train" |

**一句话概括**：这段配置让训练过程使用 MLflow 自动记录所有超参数、指标和模型文件到 `mlruns/` 目录，方便实验管理和结果对比。
